# 需求共识

> 本文档记录项目需求分析结论和已达成共识的设计决策。
> 最后更新：2026-06-30

---

## 一、项目定位

Ruminate 是一个 **AI 驱动的个人知识库系统**——通用的、开箱即用的软件产品，面向所有有知识管理需求的用户。

它不是一套 AI prompt 或 skill，而是一个完整的、可交付的软件系统。

## 二、目标用户

**初期面向所有有知识管理需求的个人用户**，典型场景包括：

- 深度研究：长期研究某个领域，读论文、文章、书籍，构建体系化知识结构
- 内容创作：写博客、做视频、写书，从海量素材中提炼观点、发现关联
- 学习笔记：系统学习课程或技能，整理笔记、建立概念连接
- 小型团队（远期）：内部知识管理，从多信息源自维护 Wiki

## 三、核心功能需求

### P0 — MVP 必须

| 功能 | 描述 |
|------|------|
| 资料摄入 (Ingest) | 接受 Markdown、纯文本、URL 等格式输入；AI 自动阅读、提取关键信息、写入 Wiki |
| Wiki 自动维护 | AI 创建/更新摘要页、实体页、概念页；自动维护交叉引用和索引 |
| 智能查询 (Query) | 基于 Wiki 内容提问；AI 综合多页面回答，带引用来源 |
| 直接检索 (Find) | 不经过 AI 的全文搜索，快速定位页面 |
| Git 原生存储 | Wiki 为 Markdown 文件的 Git 仓库，天然支持版本控制和回滚 |
| 本地优先 | 所有数据在用户本地磁盘，不依赖云服务 |
| CLI 操作入口 | `ruminate ingest/ask/find/lint/serve/config` 子命令 |
| 本地 Web 界面 | 浏览器浏览 Wiki、AI 对话、查看知识图谱 |
| 日志记录 | log.md 记录所有操作（摄入、查询、巡检）的时间线 |

### P1 — 让知识“活起来”

| 功能 | 描述 |
|------|------|
| 知识巡检 (Lint) | 自动检测矛盾、过时内容、孤立页面、缺失链接、可深入方向 |
| 增量更新 | 新资料只触发相关页面修改，无需推倒重来 |
| 完全重建 | 从原始资料从头重建整个 Wiki，消除累积偏差 |
| 问答回写 | 好的查询结果可保存为 Wiki 页面，不让洞察消失 |
| 知识图谱可视化 | 查看页面间关联、核心枢纽、知识结构全景 |
| 语义搜索 | Embedding 向量检索，支持语义匹配 |

### P2 — 进阶能力

| 功能 | 描述 |
|------|------|
| 多源连接器 | MCP/Plugin 接入网页剪辑、RSS、播客转录、Slack、会议记录等 |
| 定期复习提醒 | 间隔重复、知识健康度报告 |
| 多种输出格式 | 从 Wiki 生成幻灯片 (Marp)、图表、博客文章、学习指南 |
| 协作功能 | 团队共享、评论、建议编辑、审核队列 |
| Schema 可视化配置 | 非技术用户也能调整 Wiki 结构和规则 |

## 四、关键设计决策

### 4.1 产品形态

**CLI + 本地 Web 双形态**。CLI 做核心操作（面向开发者/power user），Web 做浏览和可视化（面向日常使用）。后期 Web 上逐渐支持更多操作，覆盖完整 Web 应用形态。

### 4.2 AI 引擎策略

- **P0 阶段**：仅使用 Ollama（本地部署），统一用于推理和 Embedding。简化初期开发。
- **P1+ 阶段**：扩展多 Provider 支持（DeepSeek、OpenAI 兼容接口）。
- **Provider 抽象**：推理接口和 Embedding 接口独立抽象，可分别配置不同 Provider。

### 4.3 存储策略

- **Wiki 内容**：纯 Markdown 文件 + Git 版本控制
- **全文检索**：SQLite FTS5（嵌入式，零运维，够用到数千页面）
- **语义搜索**（P1）：Embedding 向量检索，本地向量存储
- **索引策略**：index.md（人类可读目录）+ SQLite FTS5（机器检索）双轨制

### 4.4 检索入口

- `ruminate find "关键词"` — 直接全文检索，返回匹配页面列表（不经过 AI）
- `ruminate ask "问题"` — AI 综合回答，带引用来源

### 4.5 技术栈

| 层 | 技术 | 理由 |
|---|------|------|
| 后端 | Go | 开发者最熟悉；强 CLI 和并发支持 |
| 前端 | TypeScript + React (Vite) | AI 友好；开发者可读可维护 |
| 存储 | Markdown + Git + SQLite FTS5 | Git 原生版本控制；嵌入式全文搜索 |
| LLM (P0) | Ollama | 本地推理+嵌入，统一 Provider |
| CLI | Cobra | Go 标准 CLI 库 |

## 五、不做的事情（当前阶段）

- 不做云服务/SaaS（本地优先）
- 不做实时协作（P2 考虑）
- 不做移动端 App
- 不做 Obsidian 插件（可独立配合 Obsidian 使用，但不绑定）
